Both models trained under an identical data / token budget / schedule at d24 on a 4×H100 node. The only difference is the routing loop. The gain costs ~2.4× training compute (running the trunk twice).

How the loop works

Standard nanochat is a single-pass causal Transformer. The looped variant runs the trunk twice: pass 1 runs normally and captures its Q/K features at selected layers; pass 2 re-runs from the embeddings, and each of its layers adds a causal attention-logit prior from a chosen pass-1 layer.

Reproduce

bash runs/speedrun_baseline.sh   # stock nanochat,  tag: baseline-d24
bash runs/run_looped.sh          # routing loop on, tag: looped-d24

# full generative eval (either tag)
torchrun --standalone --nproc_per_node=4 -m scripts.chat_eval -- \
    -i sft -g looped-d24 -a "GSM8K|HumanEval"

Looped flags for scripts.base_train: --routing-loop --routing-pattern=progressive --routing-gate-init=0.05. Full methodology, cost/speed analysis, and engineering notes (FlexAttention routing kernel, distributed-optimizer fix, batched eval) are in the report.

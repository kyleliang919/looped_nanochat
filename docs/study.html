Does the loop actually help? A controlled study

A single-GPU comparison of loop / recurrent Transformer strategies, on a task chosen to have real headroom. The honest answer: the routing loop's benefit is conditional — it wins when a shallow network needs more depth, and it makes deep training far more stable, but it doesn't magically raise every ceiling. Getting here took four fairness fixes.

The fair-comparison contract

An earlier version of this study claimed “ours is the most stable strategy.” That result didn't survive scrutiny — it was a setup artifact. Getting to an honest answer required four fixes, each of which changed the conclusion:

1. Report seed means, not best-of-N — best-of-3 flatters high-variance methods.
2. Warmup + cosine LR schedule — a constant LR is hostile to deep loops (caused gradient explosions and crippled the deep baseline).
3. Match parameters, not just compute — the first study gave UT 1 tied layer (0.2M) vs ours' 6 distinct (1.19M).
4. Pick a task with headroom — screened candidates by baseline-only accuracy and pre-committed to modular cumsum, P=13 (baseline ~0.4, chance 0.077, far from ceiling) before running any loop model.

Reading it honestly

The routing loop helps when there is headroom a single pass can't reach AND the base network isn't already deep enough to capture the task (shallow + hard: +22%). When the base is already deep, extra passes don't raise the ceiling — but the near-zero-gate warm start makes training markedly more stable than the baseline or a param-matched Universal Transformer. Ours beats the param-matched UT in every configuration.

Caveats. Single synthetic task, one scale, 3 seeds — directional, not definitive. The loop's headline quality evidence remains the d24 language-modeling result (+9% CORE), where real headroom existed at scale. Earlier flawed versions of this study (unequal params, constant LR, best-seed metric, saturated task) are superseded. Full method & data: studies/README.md.
